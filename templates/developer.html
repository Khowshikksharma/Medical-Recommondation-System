Project Case Studies while Building Application

System Performance Optimization

Description: The system took too long to start because it loaded multiple large CSV files at launch.

Cause: Unoptimized file reading in Flask's startup and redundant reloading of static data.

Solution: Cached datasets in memory after first load and used lightweight databases (SQLite) for frequent queries.

Implementation: Introduced caching mechanisms that required periodic cache refreshes if data updates.

Data Imbalance Issue

Description: Rare conditions were often misclassified as common ones.

Cause: Imbalanced dataset (e.g., more "Cold" cases than "Typhoid").

Solution: Applied class weighting in SVC and augmented rare disease samples.

Implementation: Needed ongoing dataset balancing and added a disclaimer for rare conditions.

Caching Challenges

Description: Users saw outdated results due to aggressive caching.

Cause: Flask's static files cached by browsers.

Solution: Added cache-busting hashes to asset filenames and set proper Cache-Control headers.

Implementation: Required build automation for asset versioning.
